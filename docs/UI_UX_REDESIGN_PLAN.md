# 灯継ぎ UI/UX刷新設計案 — 「命脈の家譜」

> 状態: **設計提案・未実装**
> 対象: React UI全般、特に郷ホーム・戦闘・ホーム配下の各メニュー
> 実装担当想定: Claude / Fable 5
> 仕様正典との関係: 本案を採用して実装する際は `docs/GDD_v3.md` と `docs/WORKLOG.md` に確定事項・実績を反映する。

## 0. 結論

現在のゲームは機能・画像・文章の量が不足しているのではない。魅力を弱めている主因は、以下の3点である。

1. **いま重要なことと、いつでもできることが同じ強さで並ぶ**
2. **大きな作業画面と小さな選択が、ほぼ同じモーダルで開く**
3. **素材・演出・数値が、次の一手を選ぶ視線へ束ねられていない**

刷新の中心は、画面を飾り直すことではなく、プレイヤーが毎回この順で理解できるようにすることである。

> **危機や好機を知る → 今月の一手を比べる → 結果を確かめる → 次の目的が残る**

視覚テーマは既存A案「墨金の家譜」を継承する。新しい署名要素は **命脈**。家族・星契り・継足・年代記を、一本の火の線が形を変えて貫く。

- 郷: 存命者と次代を結ぶ命脈
- 星契り: 親・星神・予測される子を結ぶ命脈
- 戦闘: 同じ魔性へ重ねる継足の火脈
- 家譜: 世代を遡る時間の命脈

この1要素へ大胆さを集中し、罫線・粒子・グローの追加は抑える。

## 1. 現状確認

### 1.1 添付ホーム画面で起きていること

添付画面は「当主1人・残り14月・体力1/142・技力0/55・武功5・血珠16」という強い危機状態である。しかし、画面を見た瞬間に伝わるのは危機ではなく、広い一族欄と4つの同格な行動カードである。

| 観察 | 問題 | 改善の方向 |
|---|---|---|
| 一族欄が横に大きく、人物は左端に1枚 | 空白が状態説明に使われていない | 1〜2人時は「当主札＋血脈診断」の二面構成 |
| HP 1/142が小さなバー内だけにある | 瀕死が見落とされる | 状態語「瀕死」と静養への直接導線を出す |
| 綴は「星契りを急げ」と言うが、出立が固定で主ボタン | 推奨が競合している | 状況評価で主行動を切り替える |
| 下段メニューが背景に溶ける | 何が増えた・進んだか分からない | 新着、達成、次費用、収集率を併記 |
| 祭は無効時に全体が薄くなる | 理由まで読みにくい | 枠と見出しは残し、理由を通常コントラストで表示 |
| メニューごとの遷移形式が不統一 | 画面が開くかモーダルか予測できない | 情報量に応じて「作業画面」と「小窓」を明確に分ける |

### 1.2 実装棚卸し

ホームから開く機能は多い。

- 月を進める: 出立、星契り、祭、静養
- 郷の営み: 鍛冶と蔵、郷普請、郷を歩く、眷属、常夜百層
- 記録: 家譜、図鑑、家系図、郷の声
- 心得: 務め、家訓、手引き
- 常設: 設定

現状はホーム内に9個のbooleanモーダル状態があり、鍛冶と蔵のような複合作業も、家訓のような短い選択も同じ `.modal` で扱う。長いモーダルは最大84vhで内部スクロールし、閉じる操作が末尾にあるため、現在地と帰り道が弱い。

### 1.3 全画面で共通する摩擦

- 戻る/閉じるが画面下に置かれ、長い内容では到達にスクロールが必要
- 主要CTAも末尾にあり、比較中に確定条件が見えなくなる
- 大量一覧（星神180、魔性147基礎、装備810）が全件同じ密度で並ぶ
- 選択前、空、ロック中の画面が説明文だけになりやすい
- クリック可能な `div` があり、キーボード操作と読み上げの契約が不十分
- モーダルを閉じた際のフォーカス復帰、ESC、フォーカストラップが共通化されていない
- 月を進める不可逆行動と、無料で切替できる行動の確認強度が同じではない

## 2. デザイン原則

### 2.1 画面の一つの仕事

| 画面 | 一つの仕事 |
|---|---|
| 郷ホーム | 今月の一手を、血脈の危機と好機から決める |
| 出立 | 行き先と隊を比較し、安全に出発する |
| 星契り | 親と星神を比較し、次代の可能性を選ぶ |
| 戦闘 | 戦況を読み、次の一手と継足対象を選ぶ |
| 鍛冶と蔵 | 一人を強くする装備変更を迷わず行う |
| 郷普請 | 次に伸ばす恒久効果を比較する |
| 家譜 | 一族の歩みを読み返し、残したくなる形にする |
| 図鑑 | 未知と発見済みを整理し、次の発見欲へ繋ぐ |

### 2.2 情報の優先順位

画面内の情報は次の4階層に限定する。

1. **危機/好機** — 血脈断絶、瀕死、祭月、報酬受領可
2. **現在の決断** — 今月の行い、出立、契り、戦闘コマンド
3. **判断材料** — 数値比較、効果、予測、行動順
4. **世界観と記録** — 説明、縁起、図鑑文、演出

世界観文を削らず、判断の直前では短く畳み、選択後や詳細で十分に読ませる。

### 2.3 画面とモーダルの使い分け

#### 独立した作業画面にする

- 鍛冶と蔵
- 郷普請
- 家系図
- 家譜
- 図鑑
- 出立
- 星契り
- 戦闘/探索

基準: 複数タブ、比較、長い一覧、選択状態の保持、画面内スクロールが必要。

#### モーダル/下から出る小窓に留める

- 郷人との会話
- 眷属の随行選択
- 務めの短い一覧
- 家訓
- 手引き
- 設定
- 月を進める最終確認

基準: 1つの短い判断、10項目未満、閉じれば元の文脈へ戻る。

### 2.4 操作の強度

| 操作 | 方式 |
|---|---|
| 月を進める/大きな奉燈消費 | 効果と次月を示す確認面を1回出す |
| 出立/星契り | 現行の複数段階選択を維持し、最終CTAを固定表示 |
| 装備変更/眷属/家訓 | 即時反映＋明確な選択表示。確認は出さない |
| 安価な購入 | 1回で購入＋残高トースト |
| 高額普請/打ち直し | 差分と費用を示す確認面 |
| 画面離脱 | 入力中だけ破棄確認。閲覧だけなら即座に戻る |

## 3. 視覚システム

### 3.1 色

既存トークンを維持し、新色は増やさない。

| 名称 | 値 | 用途 |
|---|---:|---|
| 常夜 | `#0b0f1e` | 画面背景 |
| 墨藍 | `#10172e` | 作業面、カード |
| 和紙白 | `#efe6d4` | 本文 |
| 家譜金 | `#c9a86a` | 見出し、当主、確定 |
| 命火 | `#ff9d45` | 選択、推奨、命脈 |
| 朱文字 | `#e0655c` | 危機、失敗、瀕死 |

金は見出し・当主・確定へ限定する。選択と推奨は命火、危機は朱文字。属性色は属性の説明にだけ使う。

### 3.2 文字

書体は Shippori Mincho を維持する。必須情報の最小サイズを引き上げる。

| 役割 | PC | モバイル | 指定 |
|---|---:|---:|---|
| 画面題 | 24px | 21px | 700 / 0.12em |
| 節見出し | 16px | 15px | 600 / 0.16em |
| 本文 | 14.5px | 14px | 500 / 1.7 |
| 補足 | 12.5px | 12.5px | 500 / 1.55 |
| 数値 | 16px | 15px | 700 / tabular-nums |

10px以下は印、装飾、補助記号だけに限る。無効理由、費用、残寿命、HP/MPは12.5px以上。

### 3.3 余白

8pxを基準に、`4 / 8 / 12 / 16 / 24 / 32` のみを使う。

- 画面左右: PC 24px、モバイル 12px
- セクション間: 16px
- カード内: 12〜16px
- 操作間: 8px
- 主要タッチ領域: 最低44×44px

空白は残すが、「状態を説明できる空白」を優先する。1人しかいない一族欄の右側は、単なる空白ではなく血脈診断に使う。

## 4. 郷ホーム再設計

### 4.1 PC配置

```text
┌ 1年目・如月 ───────── 奉燈 583  血珠 16  武功 5 ┐
│ 血脈危機  後継なし / 当主 瀕死                     │
│ 綴「まず次代を残せ。夜藪へ戻るのは傷を癒してからだ」│
├────────────────────────────────────┤
│ [当主の大札]         │ 命脈診断                    │
│ 顔・名・残寿命       │ 存命1 / 成人1 / 子0         │
│ HP 1/142  瀕死       │ 次の危機: 後継なし          │
│                      │ [静養も必要]                │
├────────────────────────────────────┤
│ 今月の決断                                        │
│ ┌ 推奨: 星契り ────────────────────┐ │
│ │ 次代を授かる / 奉燈○ / 翌月へ進む [選ぶ]       │ │
│ └────────────────────────────┘ │
│ [出立] [祭: 今は開かない] [静養: 瀕死を回復]       │
├────────────────────────────────────┤
│ 郷の帳                                             │
│ 営み: 鍛冶 更新可2 / 普請 次費用80 / 郷 新話1      │
│ 記録: 家譜 1代 / 図鑑 12% / 家系図 1人            │
│ 心得: 務め 達成1 / 家訓 未設定 / 手引き            │
└────────────────────────────────────┘
```

### 4.2 一族欄

#### 1〜2人

左を当主/選択人物の大札、右を血脈診断にする。

血脈診断に出すもの:

- 存命/成人/幼子/懐妊中の人数
- 後継の有無
- 最短の残寿命
- 瀕死/技力枯渇/成人待ち
- 次に解決すべき問題へのボタン

#### 3〜4人

大札1＋小札3。大札は当主を既定とし、小札を押すと大札が入れ替わる。

#### 5人以上

大札1＋横スクロールの小札。全員を同じ大きさで敷き詰めない。

### 4.3 推奨行動ルール

固定で「出立」を主ボタンにしない。状態から1件だけ推奨する。

優先順位:

1. 存命1、後継なし、契り可能 → **星契り**
2. 成人0、幼子あり → **静養/月送り**（成人までの残月を表示）
3. 出立可能者の平均HPが30%未満 → **静養**
4. 祭月かつ2人以上が負傷 → **祭**
5. 奉燈が次の必須費用未満 → **出立**
6. それ以外 → **出立**

推奨の理由を1行で必ず表示する。推奨以外を禁止しない。

### 4.4 月を進める確認

祭・静養はカード押下で即実行せず、小さな確認面を出す。

```text
静養する
隊の傷を癒し、如月から弥生へ進む。
燈吾: 1/142 → 142/142
残寿命: 14月 → 13月

[やめる]  [静養して月を進める]
```

寿命が尽きる/成人/誕生/祭月など、次月に確定イベントがある場合は同じ面に予告する。

### 4.5 郷の帳

現在のghost button列を、3つの帳面へまとめる。

- **営み**: 鍛冶と蔵、郷普請、郷を歩く、眷属、常夜百層
- **記録**: 家譜、図鑑、家系図、郷の声
- **心得**: 務め、家訓、手引き

各入口は「名前＋現在値」を持つ。

| 入口 | 現在値の例 |
|---|---|
| 鍛冶と蔵 | 更新できる装備 2 |
| 郷普請 | 次の普請 奉燈80 |
| 郷を歩く | 新しい話 1 |
| 眷属 | 随行なし / 随行: 夜目 |
| 家譜 | 第7代・故人12 |
| 図鑑 | 総合34%・新着3 |
| 家系図 | 存命4・全23 |
| 郷の声 | 新着1 |
| 務め | 達成1・受領可 |
| 家訓 | 武断 / 未設定 |

PCは3列の帳面。モバイルは「営み・記録・心得」の3行accordionとする。

### 4.6 モバイル

```text
┌ 如月     奉583 珠16 功5 ┐
│ 血脈危機 / 後継なし      │
│ [当主札 横長]             │
│ 瀕死 1/142  残14月        │
│ [推奨 星契り]             │
│ [出立] [祭] [静養]        │
│ ─ 郷の帳 ─                │
│ 営み  更新2               │
│ 記録  新着3               │
│ 心得  達成1               │
├──────────────────┤
│ 郷  一族  今月  帳         │  ← 画面内アンカー
└──────────────────┘
```

下部4ボタンは別画面へのナビではなく、長いホーム内のアンカー移動に限定する。

## 5. 画面別設計

### 5.1 星契り

現行の「親選択→神一覧→神の立ち絵→遺伝予測」は良い。問題は縦に長く、最終CTAが比較対象から離れること。

改善:

- 上部に `壱 親 / 弐 星 / 参 見立て` の実ステップ表示
- PCは左に候補、右に選択中の神、下ではなく右下に遺伝要約
- 子の見立ては全6能力の細いバーに加え「伸びる2能力」「弱い1能力」を文章化
- CTAを画面下へ固定: `燈吾 × 篝火乙女 / 奉燈40 / 契る`
- 神一覧に「費用内」「未契約」「得意能力」の絞り込み
- ロック神を一覧から消さず、解禁条件を通常コントラストで表示
- モバイルは候補一覧→神詳細→見立ての3画面風スライド。長い1画面にしない

### 5.2 出立

改善:

- 上部固定の準備状況: `行き先 未選択 / 隊 0/4 / 灯100`
- 地域選択で、右側に「危険度・深さ・主・未回収要素・推奨武功」を表示
- 隊員札に現在HP、残寿命、前後列、属性相性の要点
- `自動編成` は任意で追加（HP・役割・属性から4人まで。手動変更可）
- CTA固定: `宵の森へ / 2人 / 今月を使う`
- 未達条件はCTA直上に1行。ボタン内に長文を詰めない
- 出立後の加護選択は横スクロールを発生させず、3枚＋見送るを画面幅内へ収める

### 5.3 探索

改善:

- HUDは「灯・現在地・実り・帰還」を四隅へ固定し、中央を塞がない
- 操作説明は初回だけ。以後は小休止内で見られる
- 敵影接近は視覚・音・ミニマップの3経路で予告
- 帰り火は誤操作防止の確認面。現在の持帰り量と失う可能性を比較
- 特殊床の説明は画面中央モーダルではなく、下部の短い選択面
- モバイルD-padは親指領域に固定し、キャンバスと重ならない安全余白を確保

### 5.4 戦闘 — 「火脈の戦場」

現状は戦場中央が空き、敵味方が上隅へ小さく寄り、ログと隊員札が下部を大きく占める。判断情報も「手番・属性・継足・コマンド」に分散している。

#### PC構成

```text
┌ 第2巡   [今:燈吾] → 風切り羽 → 燃え尽き人形     ┐
│                                                    │
│   [敵 大]           継             [味方 大]       │
│   [敵]        火脈: 人形へ2連       [味方]          │
│                                                    │
│ ─ 直近2行の戦況ログ ───────────────── │
├ 指令盤 ─────────┬ 一族の灯 ──────────┤
│ [攻撃:火脈を継ぐ] [技] │ 燈吾 HP/MP  手番           │
│ [防御] [逃走] [自動]   │ 次郎 HP/MP                 │
└────────────────────────────────────┘
```

#### 必須要素

- 戦場を画面高の60〜65%確保
- 敵味方を中央へ寄せ、人数が少ないほど大きく見せる
- 上段に現在から最大6手の行動順
- 中央の命脈印で継足対象、現在連数、次撃倍率を表示
- 継足対象の敵札にも「火脈/2連」を重複表示
- ログは直近2〜3行を戦場下端へ重ね、全履歴は任意で展開
- 1人編成の隊員札を横いっぱいに伸ばさない。最大幅を設ける
- 技一覧の「戻る」はスクロール外へ固定
- 対象選択中は `狙う魔性を選べ` を必ず表示し、対象だけを発光
- 敵を押す通常攻撃は維持。キーボードでもEnter/Spaceで実行可能にする
- 勝利後は戦果要約を同じ下段へ展開し、オート時は既存の自動復帰を維持

#### モバイル

- 敵を上半分、味方を中央、指令盤を下部固定
- 指令盤は2段。主要攻撃を最も大きくする
- 行動順は現在＋次3手まで
- 技は2列グリッド＋固定戻る
- 戦況ログは1〜2行。上スワイプで展開

### 5.5 鍛冶と蔵

現行は「購う・装備・打ち直し・鍛錬」が1つの長いモーダルに連続する。独立作業画面にし、4タブへ分ける。

```text
鍛冶と蔵   奉燈583 血珠16
[購う] [装備] [打ち直し] [鍛錬]
┌人物一覧────┬品一覧────────┬差分─────┐
│ 燈吾         │ 火群の刀       │ 攻 +8      │
│ 次郎         │ 月影の槍       │ 速 -2      │
└────────┴────────────┴─────────┘
             [装備する]
```

- 左に人物、中央に品、右に変更前後。モバイルは順番に積む
- `おすすめ` は現在装備との差分総合が正の品へ付ける
- 並べ替えに加え、装備可能/未装備/新着フィルタ
- 購入後に残高と取得品をトースト
- 打ち直しは失う資源と上昇予測を確認
- 810件を同時描画しない。カテゴリ/検索/ページングまたは仮想化を使う

### 5.6 郷普請

- 一覧を縦長Panel列ではなく、施設カードグリッドへ
- 各カードに現在Lv、次Lv効果、費用、最大Lvを固定位置で表示
- 「建てられる」「奉燈不足」「完成」を視覚と文言の両方で区別
- 上部に次のおすすめ1件。ただし自動決定しない
- 普請後はカード内でLvと効果を更新し、画面を閉じない

### 5.7 郷を歩く / 郷の声

- 郷人カード選択後は同じモーダル内で立ち絵と会話を右側へ表示
- 新しい台詞がある人物にだけ小さな「新」印
- 話し終えた印を付け、全員を押す作業にしない
- 郷の声は年代順/人物別を切替可能にし、新着位置へ自動スクロール

### 5.8 眷属

- 一覧、選択中の大札、効果比較の二面構成
- 随行中を先頭に固定
- 選択は即時反映し、`夜目を随行させた` と同じ語でトースト
- 空状態は「どこで出会えるか」を1行で案内

### 5.9 務め / 家訓 / 手引き / 設定

- 務め: 「受領可」を先頭、完了済みは折り畳む。目的地ボタンは維持
- 家訓: 4択をカード化。現在の家訓と効果を最上段固定。変更は即時
- 手引き: 状況検索を追加しなくてもよいが、「今困っていること」から5項目へ短縮導線
- 設定: 音量、ミュート、演出、オートに加え「月送り確認を省略」を将来追加可能

### 5.10 家譜 / 図鑑 / 家系図

#### 家譜

- `概要 / 称号 / 故人 / 年代記` の4タブ
- 戻るを上部固定
- 共有・画像保存は `残す` メニューへまとめる
- 年代記は世代・人物・出来事で絞り込めるようにする

#### 図鑑

- `土地 / 魔性 / 星神 / 宿敵` タブを上部固定
- 未知カードを大量表示せず、章/属性ごとに未知数をまとめる
- 新着だけを見るフィルタ
- カード選択で大きな詳細面。グリッド内に長文を置かない
- 大量カードはページングまたは仮想化

#### 家系図

- モーダルではなく全画面
- `当主へ戻る`、人物検索、存命だけ、代を指定
- 現在人物から祖先・子へ命脈を強調
- 詳細は右ペイン/下部sheet。図そのものを押し潰さない

### 5.11 誕生・成人・看取り・夢・結末

- 画面全体クリック進行は維持するが、下部に明示的な `次へ` も置く
- 誤タップ防止のため、選択ボタンが出た後は背景クリックで進めない
- 全体の進み具合を小さな頁印で表示
- スキップは長押し不要。確認後に章末へ移動
- 看取りは人物の顔・生涯3行・辞世を同じ視線上に置く

## 6. 共通コンポーネント仕様

Claude実装では、画面個別CSSを先に増やさず、次を共通化する。

| コンポーネント | 役割 |
|---|---|
| `ScreenShell` | 上部戻る、画面題、資源、設定、スクロール領域 |
| `ActionDock` | 画面下固定の主要CTAと条件/費用 |
| `Sheet` | 小さな選択・確認。ESC、背景閉じ、フォーカス復帰 |
| `WorkspaceTabs` | 鍛冶/家譜/図鑑などの固定タブ |
| `StatusCallout` | 危機・好機・不足理由。色だけに依存しない |
| `EmptyGuide` | 空状態＋次の行動 |
| `CompareRow` | 変更前→変更後、増減、費用 |
| `LiveBadge` | 新着/達成/更新可。0なら非表示 |
| `LifeThread` | 命脈の表示。ホーム/契り/戦闘/家譜で形を変える |

### 6.1 戻る/閉じる語彙

- 独立画面: `郷へ戻る`（左上）
- モーダル: `閉じる`（右上）
- 選択取消: `選ばず戻る` または `やめる`
- 実行取消: `実行しない`

同じ操作は同じ語を使う。

### 6.2 フォーカス

- 開く前の要素を保存し、閉じたらそこへ戻す
- モーダル中はTabを内部へ閉じ込める
- ESCで閉じる。ただし不可逆処理中は無効化し理由を表示
- クリック可能な要素は `button` を使う。やむを得ない場合はrole/tabIndex/Enter/Spaceを実装

### 6.3 状態保持

- タブ、並べ替え、フィルタ、選択人物、スクロール位置を画面離脱まで保持
- モーダルを閉じてもホームのスクロール位置を変えない
- エラー後に入力・選択を消さない

## 7. アクセシビリティと操作性

- WCAG 2.2 AA相当のコントラスト。既存 `--ember-text` を危機文に使う
- 主要文字12.5px未満禁止
- タッチ44px以上
- `:focus-visible` を全操作へ
- 選択は色＋太枠＋印＋文言
- 無効ボタンの理由を常時表示し、title属性だけにしない
- `prefers-reduced-motion` とゲーム内演出軽減の両方を維持
- ダメージ・危機・戦果は動きを止めても情報が残る
- 画面読み上げ用に、資源変化・装備変更・戦闘結果は短いlive regionへ
- 横スクロールは地図/家系図など意味がある場所だけ。通常モーダルでは発生させない

## 10. 成功指標

定量:

- ホーム到着から推奨行動を選ぶまで3操作以内
- 鍛冶で人物選択→装備比較→装備まで4操作以内
- 出立準備で地域/隊/CTAが常に画面内または固定領域で確認可能
- 戦闘で現在手番、次手、継足対象を1画面で確認可能
- PC/モバイルで意図しない横スクロール0
- キーボードだけで全主要フロー完了
- console error 0、data検証0件失敗

定性:

- 「何をすべきか」だけでなく「なぜ今それか」が分かる
- メニューを開く前に、そこに新しいものがあるか分かる
- 閉じ方/戻り方を覚え直さない
- 1人編成・1人一族でも画面が空疎に見えない
- 機能量が一覧の密度ではなく、選択の期待へ変換されている
