# 灯継ぎ UI/UXブラッシュアップ修正指示 — Claude/Fable 5向け

作成日: 2026-07-11  
対象: M18/M19実装後の画面検収とブラッシュアップ  
実装担当: Claude/Fable 5  
正典: `docs/GDD_v3.md` §8.5〜§8.8

## 0. この指示の目的

物量を見せるだけの画面から、次の判断が短時間でできる画面へ直す。

1. 何を選べるか
2. 選ぶと何が変わるか
3. 現在どこまで準備できたか
4. 次に何を押すべきか

見た目だけを整えるのではなく、比較・選択・取消・復帰の摩擦を減らす。既存のゲームロジックは変更しない。

## 1. P0 — 「隊を組む」の横長1枚カードを解消する

### 1.1 原因

`src/index.css` の共通指定:

```css
.exp-party { display: flex; gap: 8px; flex-wrap: wrap; }
.exp-party .char-card { flex: 1; min-width: 150px; }
```

候補が1人でも `flex: 1` により行全体を占有する。`.exp-party` は星契りや出立後の隊表示でも使われるため、共通指定を直接変更しない。

### 1.2 必須レイアウト

`src/ui/Expedition.tsx` の編成部分だけに専用クラス/専用コンポーネントを追加する。

```text
隊を組む（1/4）
┌ 隊列 ─────────────────────────────┐
│ [1 橙吾] [2 空き] [3 空き] [4 空き] │
└───────────────────────────────────┘

候補
[橙吾] [候補2] [候補3] [候補4]
```

- PC 1100px以上: 候補を4列。1人でも第1列の幅だけを使う。
- 641〜1099px: 候補を2列。
- 640px以下: 1列のコンパクト横カード。モバイルで全幅になること自体は許容するが、高さを抑える。
- PCの候補カード上限は280px程度。親の残り幅へ伸ばさない。
- 5人目以降は次行へ自然に折り返す。横スクロールは禁止。
- 4つの隊列枠は常時表示し、空き枠を明示する。

実装候補:

```css
.depart-party-grid {
  display: grid;
  grid-template-columns: repeat(4, minmax(0, 1fr));
  gap: 12px;
  align-items: start;
}

.depart-party-grid > * {
  min-width: 0;
}
```

ブレークポイントは既存の設計トークン/メディアクエリへ合わせる。`src/index.css` の `.exp-party` を一括修正して他画面へ波及させない。

### 1.3 候補カードの情報量

現行 `CharCard` の大きなHP/MPバーと6能力の横一列は、1人しかいない時に「人物詳細画面」のように見える。編成用カードは専用のコンパクト表示にする。

常時表示:

- 肖像、名前、世代、属性
- 年齢/月齢、残り寿命の目安
- HP/MPの短いバーと数値
- 役割/家業、状態異常、出立警告
- 選択中なら隊列番号1〜4とチェック印

詳細へ退避:

- 力・体・技・速・心・運の全数値
- 長い人物説明
- 装備詳細

カードを選択した時は右側詳細面、popover、または下部sheetで全能力を確認できるようにする。「情報を消す」のではなく「比較面から詳細面へ分ける」。

### 1.4 操作要件

- カード全体をbutton相当とし、マウス、Enter、Spaceで選択/解除できる。
- `aria-pressed` で選択状態を伝える。
- 色だけに頼らず、チェック印と隊列番号を表示する。
- 4人選択済みで別候補を押した場合は黙って入れ替えず、「隊は四人まで」と近接表示する。
- 二度押しで解除できる。
- 選択順と隊列番号は一致させる。解除後は番号を詰める。
- 出立不可の理由をCTA近くに表示する。ボタンを押してから初めて理由を出さない。
- タップ領域44px以上、本文12.5px未満にしない。

### 1.5 空状態/境界状態

- 成人が0人: 空のグリッドではなく、成人条件と次に取れる行動を案内する。
- 成人が1人: PCで横長にしない。隊列の第1枠＋候補第1列に収める。
- 成人が2〜4人: 常に同じカード幅を保つ。
- 成人が5人以上: 折り返し後も選択済みを先頭固定しない。並びが突然動かないようにする。
- HPが低い/寿命が近い: 朱色だけでなく警告文と印を付ける。

### 1.6 変更対象

- `src/ui/Expedition.tsx`
- `src/ui/depart_m18.css`
- 必要な場合のみ編成専用コンポーネントを `src/ui/` 配下に追加
- 共通 `CharCard` を変える場合は `Pact.tsx` と出立後の隊表示を必ず回帰確認

### 1.7 受入条件

- 1280×720、成人1人で候補カードが編成領域の横幅いっぱいにならない。
- 1280×720、成人4人で4枚が同じ幅で1行に収まる。
- 成人6人で5、6人目が2行目へ折り返し、横スクロールがない。
- 390×844で文字切れ、横はみ出し、固定CTAとの重なりがない。
- マウスなしで4人選択→1人解除→再選択→出立まで操作できる。
- 1人/2人/4人/6人、成人0人、HP1、寿命警告ありを画面確認する。

## 2. P0 — 装備の区分・レアリティ・比較を理解できるようにする

### 2.1 データの軸を混同しない

次の5軸を別々に扱う。

- 装備部位: 武器 / 防具 / 装飾
- 品質: 粗末 / 良品 / 名品 / 秘宝 / 神器
- 希少度: common / uncommon / rare / epic / legendary
- 入手由来: 店 / 宝箱 / ボス / 家宝 / 神授
- 継承: 何代受け継いだか

現行 `shopTier` をそのまま希少度として見せない。まずGDDに名称・効果・色・入手率を定義し、セーブ互換を守るoptionalフィールドで追加する。

### 2.2 一覧表示

- 各品に武器/防具/装飾の字印を付ける。
- 品質/希少度は色＋枠模様＋文字で示す。
- 現在装備中、装備可能、条件不足を一目で区別する。
- 装備比較は「攻撃+3、防御-1、速+2」の差分を表示する。
- 装飾品の `statBonus` を比較対象に含める。攻防だけの比較で「同等」と誤表示しない。
- 鍛冶の大量一覧は初期状態でカテゴリ別。検索、部位、希少度、装備可能のみの絞り込みを設ける。

## 3. P0 — 神立ち絵の欠落をフォールバックで隠さない

- 冥麗星姫と焔后は同じ絵ではなく、参照画像欠落により同じ `✦` フォールバックが見えている状態として扱う。
- `docs/GOD_ART_AUDIT_2026-07-11.md` と `docs/VISUAL_ASSET_AUDIT_2026-07-11.md` の不足一覧を正本とする。
- 画像欠落時は本番カードに巨大な共通記号を出さず、「絵姿準備中」の明示と属性別仮シルエットへ退避する。
- 通常絵とMAX絵は同一人物性を保つため、通常絵の承認後に参照画像からMAX差分を作る。
- 星契り一覧で同じフォールバックが2件以上見えたら出荷不可。

## 4. P1 — モーダル/Sheetを共通操作に統一する

- 画面外クリックで閉じる。
- Escapeで閉じる。
- 内側クリックは閉じない。
- 開いた時に見出しまたは最初の操作へフォーカスする。
- 閉じた時に起点ボタンへフォーカスを戻す。
- 背景スクロールを止める。
- `role="dialog"`、`aria-modal="true"`、ラベル関連付けを付ける。
- 保存中、戦闘結果、取り返しのつかない確定だけは誤閉鎖を防ぐ。外側クリック時に確認を出すか無効化理由を示す。

新規モーダルを増やさず、可能な限り `src/ui/layout/shell.tsx` の `Sheet` へ寄せる。旧raw modalは一覧化して順次置換する。

## 5. P1 — 「郷を歩く」を実際の移動空間にする

カード4枚のモーダルを最終形としない。PixiJSの小規模な郷マップを実装する。

- 中央: 大燈籠
- 周囲: 星契りの祠、鍛冶と蔵、豆腐屋、出立門
- NPC: 家族、鉄造、タネ婆、小鈴、松吉
- WASD/矢印/タップ移動、近接時に会話/施設名を表示
- 初回訪問後はミニマップまたは施設一覧から即移動可能
- 郷内移動では月を消費しない
- NPCの頭上に新着/依頼/会話差分を色だけでなく印付き表示
- 移動が面倒なプレイヤー向けに「すぐ行く」を常設する

歩くことを強制ロード時間にせず、世界観を味わう経路と最短操作の両方を残す。

## 6. P1 — 出立地図・ボス・ダンジョンの予告を強化する

詳細は `docs/VISUAL_RECOVERY_DUNGEON_PLAN.md` を実装正本とする。

- 出立画面を開いた時点で最前線を選択済みにする。
- 地域画、地域名、短い情景、危険度、推奨武力、主な報酬、ボスの輪郭をクリック前から見せる。
- 地図ノードだけで終わらせず、選択地域の署名ランドマークと危険兆候を右面に表示する。
- ダンジョン内で痕跡を3段階集め、ボス属性/危険技/見切りへ接続する。
- 40地域を4テーマの色替えだけにせず `RegionVisualProfile` で地形、光、粒子、ランドマーク、環境音を差別化する。
