## 1. 結論

現状の主要問題は、画面の数やデータの不足ではない。次の5点である。

1. 多くの画面が同じ濃紺パネルと小さな文字の一覧になり、場所の違いが感じられない。
2. 閲覧と実行が同じカードのクリックへ結びつき、比較前に購入・装備・鍛錬が確定する。
3. 推薦、警告、綴の台詞、次の一手が重複し、重要な一件がかえって埋もれる。
4. 郷普請、季節、地域制覇、一族の変化が、歩ける郷の景色へ反映されない。
5. PCの右ペイン前提や長い横並びがモバイルへそのまま積まれ、詳細へ到達しにくい。

目標体験を「郷で暮らしを見る → 一つ判断する → 景色と家譜に結果が残る」の三拍子に定める。数字を増やすのではなく、判断と余韻を強くする。

## 2. 既に良い部分 — 維持すること

以下は後退させない。

- ScreenShell、Sheet、ActionDock、EmptyGuide、CompareRowの共通化。
- SheetのEscape、背景クリック、フォーカス復帰、背景スクロール停止。
- 装備の部位・品質・希少度・入手由来・継承の5軸。
- 装備比較で攻防と六能力を別軸表示し、パレート優越だけを「薦」とする規則。
- 出立編成の4固定枠、選択順、1人でも横幅一杯へ引き伸ばさない候補札。
- 郷の歩行と「すぐ行く」の併設。歩行を強制ロードにしない方針。
- 図鑑の未知項目集約、地域縁起の段階開示、宿敵の記録。
- 神立ち絵の通常/MAX差分と属性別フォールバック。
- 戦闘のワンタップ攻撃、オート既定、全buttonへの共通UI効果音。
- streak損失、FOMO、通知誘導、行動力待ちを採用しない方針。

## 4. 全画面の共通設計

### 4.1 一画面一判断

- 画面下の強いCTAは原則一つ。
- 戻る、閉じる、共有、画像保存は主CTAと同じ強度にしない。
- 推薦は一画面一件。同じ行き先を綴、警告、診断、札の四箇所へ重複させない。
- 推薦には必ず根拠を一行で添える。単なる最安を「おすすめ」と呼ばない。
- unavailableな項目も情報は閲覧可能にし、実行ボタンだけを無効化する。

### 4.2 「見る」と「実行」を分ける

一覧カードの第一操作は選択または詳細表示とする。購入、契り、普請、血珠消費は詳細面のCTAから行う。

推奨フロー:

1. 一覧で候補を選ぶ。
2. 同じ画面の詳細ペイン、モバイルではSheetで差分を見る。
3. 費用と結果が見えるCTAを押す。
4. 高価または月を進める操作だけ最終確認する。
5. 結果をトーストだけで終わらせず、変わった場所を一度見せる。

安価な購入まで毎回二重確認にしてはならない。詳細面の「購う」自体を意図的な確定操作とし、残奉燈が大きく変わる時だけ確認を追加する。

### 4.3 PCとモバイルの詳細面

- 769px以上: 左が一覧、右がsticky詳細のmaster/detail。
- 768px以下: 一覧は一列または二列、項目選択で詳細Sheet。
- Sheetを閉じると、元の項目とスクロール位置へ復帰する。
- 戻った時にタブ、検索、フィルタ、選択人物、スクロール位置を失わない。
- 固定ActionDockと固定ホームナビを同じ画面で二重に出さない。

### 4.4 画面の場所性

全画面へ背景画像を大きく敷く必要はない。上部25〜35%に場所の絵、下部に作業面を置き、暗いグラデーションで接続する。

- 鍛冶: 火床、槌、棚。橙の一点光。
- 見世: 品棚と包み。柔らかな灯。
- 星契り: 祠、注連縄、星輪。青紫と金。
- 普請: 設計図、木組み、完成外観。墨と木の色。
- 出立: 門と道の奥。冷たい藍。
- 家譜/図鑑: 紙面と索引。景色画像は選択項目の証拠として使う。

同じ全面パネル、同じ太さの金枠、同じグローを増やす対応は禁止する。

## 5. 郷ホーム

### 5.1 現状の問題

- 血脈危機、綴の一言、血脈診断「次の一手」、推薦理由、推薦ActionCardが同じ行き先を繰り返す。
- 常時4枚の月行動を同じ面積で見せるため、選べない祭も毎月大きな空間を使う。
- 家族、一族診断、今月の決断、郷の帳がすべて大枠パネルで、視線の起点が弱い。
- モバイルでは行動説明を大きく省略し、決断の意味が薄くなる一方、固定4アンカーが画面下を占有する。
- 日参りと今日の御題が実時間の日課として前面に出る。家族の月次物語より強くしてはならない。
- 手引き、家訓、郷の声、眷属は動くが、一覧の検索・比較・新着へ不足がある。

### 5.2 新しい構成

上から次の順へ固定する。

1. 暦・奉燈・血珠・武功。数値は現状維持。
2. 「今月の要」。危機または好機を一件だけ表示し、綴の台詞と根拠を統合。
3. 「家の灯」。選択中人物の顔、HP/心労/残り灯、次に起きる家族イベント。
4. 「今月の決断」。第一候補を幅広の主札、残りを三つの小札。
5. 「郷の帳」。営み・記録・心得。新着と現在の課題がある入口だけ強調。

血脈診断の数値は「家の灯」へ吸収する。危機時に同じCTAを複数箇所へ出さない。

### 5.3 月行動の表示

- 推薦札: 見出し、なぜ今か、月消費、主な結果。
- 代案札: 名称、月消費、結果一行。
- 選べない祭: 大カードにせず「次の祭まで二月 / 奉燈不足」等の一行へ縮める。
- 静養: 何人がどれだけ回復するかを確認前から要約。
- 星契り: 子が生まれる時期と未契約の成人数を要約。
- 出立: 前回地域、未回収の痕跡、負傷者数を要約。

### 5.4 補助メニュー

- 手引き: 「今の画面」「家族」「探索」「戦闘」の検索可能な索引へ。現在発生中の危機から該当項目を開ける。
- 郷の声: 人物、未読、出来事の種類で絞る。未解禁の？？？は総数だけを出し、長い空欄一覧にしない。
- 眷属: 現在随行中を上部へ固定し、各効果と今回地域で効くかを比較。切替後に閉じなくても結果が分かる。
- 家訓: 選択前後の効果、現在の家訓、変更に費用/制限がないことを明記。
- 今日の御題: 郷の帳内の二次要素とし、未達で損をする印象を出さない。

### 5.5 受入

- 危機時でも同一行き先の強いCTAは一つ。
- 1280×720で、暦、今月の要、家の灯、第一候補が初期表示内に入る。
- 390×844で固定ナビが主CTAやSheetと重ならない。
- 日参り通知はbuttonで閉じられ、EscapeやSheetのフォーカス順を壊さない。

## 6. 郷を歩く

### 6.1 現状の問題

- VillageEngine.layoutは22×11全体を常にfitする。歩行者を追うカメラがなく、人物と建物が小さい。
- MAP、NPC_SPOTS、家族位置が固定で、再訪時の変化が少ない。
- facilities、季節、regionsClearedがエンジンへ渡されず、投資と冒険が郷の景色に残らない。
- 建物は手続き描画の矩形中心で、鍛冶、祠、豆腐屋、門の記憶差が弱い。
- モバイルは上部の5つの即移動ボタンと下部D-padで、歩ける面が狭い。
- 会話は歩行を止めず残るため、人物から離れても会話帯が居続ける。

### 6.2 カメラ

- 標準は主人公追従。PCで横13〜16タイル、390pxで横8〜10タイルを目安に見せる。
- マップ端ではworld外の空白を見せないようクランプする。
- タップ移動時は目的地へ最大1.5タイルだけlook-aheadする。
- 「見渡す」を押している間だけ全景fitへ移り、離すか施設を選ぶと追従へ戻る。
- reduce motion時は補間を短くし、急なzoom演出を行わない。

### 6.3 景色へ状態を反映

VillageEngineOptsへ次を追加する。

- seasonIndexまたはseasonBand
- facilitiesのLv
- regionsCleared件数またはprogressBand
- family人数帯

最低限の反映:

- 湯屋: Lv0は空地、Lv1小屋と湯気、Lv2露天と客、Lv3灯列と広い湯煙。
- 書庫: Lv0木箱、Lv1小書庫、Lv2増築、Lv3二階と紙灯。
- 社: Lv0小祠、Lv1鳥居、Lv2星輪、Lv3参道灯。
- 物見櫓: Lv0足場、Lv1低櫓、Lv2見張り灯、Lv3高櫓と遠見火。
- 春は花片、夏は蛍、秋は落葉、冬は雪。常時大量particleにせず静的decalを主とする。
- 地域制覇10/20/30件で門の旗、土産、旅人の影を段階追加。
- 一族が増えれば家族の姿が郷の別地点にも現れる。

普請はカード上の数値だけでなく、郷へ戻った時に必ず一度「ここが変わった」と分かること。

### 6.4 導線

- PCの「すぐ行く」は一列のままでもよいが、現在地と未読を示す。
- 390px以下は「すぐ行く」一ボタンから下部Sheetを開く。常設5ボタンを舞台の上へ積まない。
- 建物へタップすると最寄りまで自動歩行し、近接で「入る」。もう一度のタップ要求はしない。
- 会話中は移動入力を弱め、人物から一定距離離れたら会話を閉じる。
- D-padは上/下/左/右のaria-labelを持ち、44px以上。タップ移動端末では視覚強度を下げる。

### 6.5 受入

- 主人公の見た目高さは通常表示で56〜88px。
- 1280×720と390×844の両方で、主人公、近い施設、進める道が一目で分かる。
- Lv0とLv3の4施設を静止画で判別できる。
- 季節4種を色調だけでなく、小物または地面差分でも判別できる。
- 「すぐ行く」は2操作以内、歩行は月を消費しない。

## 7. 鍛冶と蔵

### 7.1 現状の問題

- 購うタブでは内部のselCharが当主へ自動決定され、薦を計算するが、誰への推薦か見えない。
- 商品は押すと即購入、蔵の品は押すと即装備される。詳細を読みたいだけでも実行になる。
- 購う一覧には差分数値がなく、薦印だけで判断させる。
- 装備対象の人物帯は名前だけで、顔、役割、現在装備、危機状態が分からない。
- 不足品や打ち止め品はdisabledとなり、なぜ欲しいか、次値がどうなるかを見られない。
- 4タブのうち血潮鍛錬だけ人物の遺伝資質を扱い、鍛冶/蔵との意味が離れる。
- 810品の一覧でアイコン、部位、品質、希少、由来、継承、攻防が同じ小札へ積まれ、商品自体の魅力が弱い。

### 7.2 情報構成

タブ内部keyは維持してよい。表示名を次へ変更する。

- buy: 見世
- equip: 蔵と装い
- reforge: 打ち直し
- train: 血潮鍛錬

全タブ上部に人物コンテキスト帯を置く。

- 見世: 「比較する相手」として任意。初期値が当主なら明記。
- 蔵と装い/血潮鍛錬: 実行対象として必須。
- 打ち直し: 装備者または蔵の所在を表示。

人物帯は顔、名前、家業、HP/心労警告、武防飾の略装備を含む。横長一人札へ戻さず、最大4〜6人の横スクロール札とする。

### 7.3 商品と装備のmaster/detail

一覧札に残す情報:

- 画像
- 名称
- 武/防/飾の字印
- 攻または防の主値
- 希少度
- 装備中/所持数/形見
- 比較対象に対する主な差分一つ

詳細面へ逃がす情報:

- 大きい装備絵
- 品質、希少度、入手由来、継承の説明
- 攻防と六能力の全差分
- 現在装備と交換後の総変化
- 誰が装備中か
- 価格と購入後残高
- 購う、装備する、打ち直すのCTA

カード自体は選択だけにする。モバイルは詳細Sheetを開く。

### 7.4 実行規則

- 購入: 詳細面CTAで確定。残奉燈の25%以上を使う場合だけ確認Sheet。
- 装備: 詳細面で交換前後を見せ、装備CTA。実行後5秒の「元へ戻す」をトーストに出せるなら採用。
- 打ち直し: 現行確認Sheetを維持し、不足品でもSheetは閲覧可能。
- 血潮鍛錬: 能力、回数1〜上限、増加量、総血珠、残血珠を出して一括確定。連打消費を廃止。
- 入手/消費のコア計算は既存関数を単一情報源とし、表示用に式を複製しない。

### 7.5 検索と大量一覧

- 検索結果件数と「条件をすべて外す」を常設。
- 部位、希少度、由来、装備可能、現在より上がる軸を絞り込み可能にする。
- フィルタ0件時は現在条件を列挙し、解除ボタンを出す。
- 50件ずつの追加表示は維持可能。追加後にスクロール位置を飛ばさない。
- 小さなチップを増やし続けず、一覧では代表二軸、詳細で全軸を見せる。

### 7.6 受入

- 購うタブの「薦」が誰に対するものか必ず見える。
- 情報を見るだけで購入、装備、鍛錬が発生しない。
- 奉燈/血珠不足でも品の詳細と次値を読める。
- 390pxでフィルタが画面の半分以上を常時占有しない。
- 形見、主討伐品、店売り神器など5軸が矛盾なく詳細表示される。

## 15. 共通コンポーネントとA11y

### 15.1 WorkspaceTabs

実装要件:

- 各tabへid、aria-controls。
- 対応panelへrole=tabpanel、aria-labelledby。
- 選択中だけtabIndex=0、他は-1。
- Left/Right、Home/Endでフォーカス移動。Enterまたは自動選択は既存操作感に合わせて一方式へ統一。
- モバイルで折り返して二段になる場合、横スクロールの一段を優先。タブ順を視覚順と一致させる。

### 15.2 Sheet/Dialog

- 背景クリックは閉じるだけで、決定を実行しない。
- 高リスク確認も背景クリックでキャンセル可能。未確定内容を失う場合は警告する。
- Sheet titleをaria-labelledbyで関連付ける。
- mobile Sheetへdrag handleを付けてもよいが、スワイプだけを唯一の閉じ方にしない。

### 15.3 文字と操作

- 本文14px以上、補助12.5px以上を基本。11pxは装飾/重複情報だけ。
- 主要button 44×44px以上、D-pad 48px以上。
- 色だけで希少度、警告、選択を表さず、字印/枠/文言を併用。
- focus-visibleをすべてのbutton、input、tab、カード選択へ。
- 画像altは情報画像だけ付け、背景/枠は空alt。
- live regionは短い結果だけ。操作buttonをrole=statusの中へ置かない。

### 15.4 状態保持

画面から詳細Sheetを開閉しただけで以下を失わない。

- 選択人物。
- 選択タブ。
- 検索/フィルタ。
- 一覧スクロール。
- 選択地域/神/装備。

ゲームセーブにUIの一時状態を混ぜる必要はない。画面コンポーネント内または専用の一時UI stateでよい。
