# 継続動機・快適性・素材拡充計画

**策定日**: 2026-07-11  
**対象**: M18 UI/UX刷新後の次工程  
**実装担当**: Claude / Fable 5  
**本書の役割**: 「もう一月だけ遊びたい」を生む仕組みと、疲れず戻れる操作性、神絵の拡充順を一つにまとめる。

## 1. 結論

『灯継ぎ』の継続動機は、無期限の日課や取り逃しではなく、次の三つで作る。

1. **家族への未完の約束** — 次に何を成し遂げたいかが一文で残る。
2. **神との関係の変化** — 再会するほど台詞・表情・装い・加護の意味が変わる。
3. **世代をまたぐ予告と回収** — 今代の判断が、次代の人物・郷・宿敵へ目に見えて残る。

物量はすでに十分ある。追加素材は「数を見せる」ためではなく、重要な再会・節目・結果を記憶に残すために使う。

## 2. 健全な「中毒性」の定義

本作で目指すのは、止めづらくする依存設計ではなく、**自分の意志で続きを見たくなる継続性**である。

採用する:

- 遊ぶたびに小さな物語が一つ完了する。
- 終了時に次の楽しみが一つだけ見える。
- 休んでも不利にならず、前回の意図をすぐ思い出せる。
- 同じ行動でも、家族・神・宿敵の文脈が変化する。
- 長期目標は複数あるが、今やることは一つに絞れる。

採用しない:

- ログイン連続日数の途切れによる損失。
- 期間限定報酬、時限イベント、FOMO。
- 行動力回復待ち、通知での呼び戻し。
- 有償または疑似有償のランダム抽選。
- 結果確率を隠して反復を煽る仕組み。
- 月送りのたびに複数の報酬画面を重ねること。

## 3. 時間別の遊びの輪

| 時間 | プレイヤーの約束 | 画面に必要なもの | 終了感 |
|---|---|---|---|
| 30秒 | 前回何をしていたか分かる | 「前回の灯」1文＋再開CTA | 迷わず戻れる |
| 3分 | 一月の判断が一つ進む | 今月の推奨、予告、結果差分 | 一月分の区切り |
| 10〜15分 | 一回の出立に起伏がある | 目的、危険、発見、帰還まとめ | 小さな冒険の完結 |
| 30〜60分 | 家族か神との関係が変わる | 成長、契り、看取り、宿敵進行 | 一章分の余韻 |
| 数世代 | 一族だけの歴史が形になる | 家譜、遺品、家訓、因縁の回収 | 語りたくなる物語 |

## 4. P0 — 最初に入れる継続システム

### 4.1 「前回の灯」再開札

ゲーム終了時または大きな画面遷移時に、最後の意図を自動保存する。

表示例:

> 前回の灯 — 燈吾は後継を得るため、月夜見の大神との契りを目指していた。

要件:

- 郷ホーム上部に一枚だけ表示する。
- CTAは文脈に合う一つだけ。「星契りへ」「出立準備へ」「家譜で確認」など。
- 完了・不可能になった札は自動で閉じ、理由を一度だけ伝える。
- プレイヤーが手動で「この目的を覚えておく」を選べる。
- 日付や連続起動とは結びつけない。

受入:

- 7日後に再開しても10秒以内に次の操作を決められる。
- セーブ互換を壊さず、保存値がない旧セーブでは表示しない。

### 4.2 「家の火種」三段目標

長期目標を一覧で大量表示せず、同時に三つまで掲げる。

- **今月の火種**: 1〜3分。例「負傷者を休ませる」。
- **今代の火種**: 10〜60分。例「宿敵・濡れ女を退ける」。
- **家の火種**: 数世代。例「水脈の神と三代続けて縁を結ぶ」。

要件:

- 達成報酬は小さな数値だけでなく、家譜文・称号・装飾・台詞を優先する。
- 破棄や差し替えに罰を付けない。
- 自動候補は現在の危機・未発見・家系傾向から三件以内。
- ホームでは進行中の一件だけを見せ、詳細は郷の帳へ置く。

### 4.3 「次月の兆し」

月送り確認に、確定情報と曖昧な予兆を一つずつ出す。

例:

- 確定: 「来月、篝吾が成人する」
- 予兆: 「北の山道に黒い火の噂がある」

要件:

- 数値損得を全て開示する必要はないが、不可逆な危険は隠さない。
- 同じ予兆を何度も出さない。
- 月送り後は差分を一画面にまとめ、個別ポップアップを連打しない。

### 4.4 帰還の一枚絵「今回変わったこと」

探索帰還後は戦利品一覧だけで終えず、最大三件に圧縮する。

優先順:

1. 人物の生死・成長・傷。
2. 神・宿敵・地域との関係変化。
3. 新発見・重要装備。

各行から関連画面へ一回で移動できる。確認後の主CTAは「郷へ戻る」一つ。

## 5. P1 — 神を再会したくなる存在へ変える

### 5.1 縁の三幕

各神に共通する三つの関係節目を設ける。全180柱へ長編を用意せず、短い差分を共通構造で量産する。

| 節目 | 解放 | 変化 |
|---|---|---|
| 邂逅 | 初契り | 通常立ち絵、第一印象、一族への呼び方 |
| 親縁 | 縁3 | 再会台詞、家族一人への言及、背景小物差分 |
| 深縁 | 縁MAX | MAX立ち絵、固有短話、家譜印、余韻台詞 |

大切なのは「好感度が5になった」ではなく、**神が一族を覚えた証拠**を見せること。

実装上の要点:

- 現行 `godAffinity` と `GOD_MAX_AFFINITY` を活用し、別の重い進行値を増やさない。
- 神ごとに一族への呼称を一つ持たせる。
- 同じ神と世代をまたいで契った時、前代の名前を一度だけ呼ぶ。
- MAX絵がない場合は通常絵へ安全に退避する現行仕様を維持する。

### 5.2 神縁の余韻

契り直後に報酬列を出すだけでなく、次の再会を一文だけ予告する。

例:

> 月夜見の大神は、家譜の余白にまだ読めぬ夜の印を残した。

この印は郷の帳に残り、条件が近づくと文が変わる。時限ではない。

### 5.3 選抜カットイン

全180柱のカットインを作らない。以下に絞る。

- 極ツ星10柱。
- 物語の鍵となる神2〜6柱。
- 初めて縁MAXへ到達した神の属性共通演出。

通常の神授は立ち絵＋属性紋＋短い画面演出で成立させる。カットインは希少性を守る。

## 6. P1 — 宿敵と世代の引き

### 6.1 「宿敵の傷跡」

敗北・撤退・取り逃しを単なる失敗にせず、次の対面へつなぐ。

- 敵に一族への反応が一つ増える。
- 家譜に傷跡が記録される。
- 次の遭遇条件が少しだけ読みやすくなる。
- 再戦時に前回との差分を戦闘開始前の一文で示す。

敗北を得にする必要はない。**失敗にも物語が残る**ことが目的。

### 6.2 「世代の問い」

当主の成人・継承・看取りの前後に、その世代の問いを一つ選ぶ。

例:

- 「強さを遺す / 人を遺す / 名を遺す」
- 「神を信じる / 郷を信じる / 血を信じる」

選択は大きな数値分岐にせず、家訓候補、事件の文面、後継の初期傾向、家譜の見出しへ薄く広く反映する。次代で一度回収し、永遠に積み上げない。

## 7. 快適に遊ぶための必須要件

### 7.1 操作

- 主要CTAは各画面一つ。危険操作と隣接させない。
- 戻る操作を画面上部とESC/端末戻るで統一する。
- 長い一覧でも主CTAと戻るを固定する。
- 同じ月に何度も開く画面は、直前のタブ・絞り込み・スクロール位置を保持する。
- 無効ボタンは理由を近くに表示し、押して初めて理由が出る設計にしない。
- 月送り、契り確定、出立、遺品破棄など不可逆操作だけ確認する。

### 7.2 情報量

- ホームで同時に強調する危機は最大二件。
- 新着バッジは「未読数」より「何が増えたか」を優先する。
- 数値変化は `42 → 47 (+5)` のように前後を併記する。
- 装備比較は総合値だけでなく、変わる能力を上から三件まで表示する。
- 専門語は初回だけ短い説明を出し、以後は長押し/ヘルプへ退避する。

### 7.3 失敗回復

- 戦闘外の選択は確定前なら戻せる。
- 自動装備や一括処理には結果プレビューを付ける。
- セーブ中は二重操作を止めるが、長い全画面ロックはしない。
- 復帰時に壊れた座標・存在しない対象・終了済み目標を安全に解決する。

### 7.4 演出

- 通常操作の演出は150〜250ms、重要な節目は600〜1200msを目安にする。
- 同時に動かす主役は一つ。背景粒子は主情報より弱くする。
- `prefers-reduced-motion` では移動量を抑え、ダメージ数値や状態変化は消さない。
- 演出スキップは長押しではなく一回入力で可能にする。

### 7.5 モバイル

- 390×844で固定CTAが本文・端末safe-areaと重ならない。
- タップ領域44px以上。
- 一覧は横スクロールへ逃げず、比較面をsheet化する。
- 戦闘の対象選択、探索D-pad、帰り火を片手で操作できる。

## 8. 神絵の拡充方針

監査の詳細は `docs/GOD_ART_AUDIT_2026-07-11.md`、制作規格は `assets_src/GOD_ART_STYLE_GUIDE.md` を参照する。

現状:

- 神データ: 180柱。
- 通常立ち絵: 120 / 180。**60柱不足**。
- 縁MAX立ち絵: 120 / 180。**同じ60柱が不足**。
- 神カットイン: 24 / 180。現行manifestが先頭24柱だけを対象にした仕様。
- manifestの通常立ち絵登録: 96 / 180。実ファイル120枚と不一致。

制作順:

1. **P0-A 序盤20柱**: rank 1の不足。初回数時間の代替表示をなくす。
2. **P0-B 終盤10柱**: rank 4の不足。最高位解放のご褒美を成立させる。
3. **P1 中盤15柱**: rank 2。
4. **P2 上位15柱**: rank 3。
5. 上記60柱の通常絵承認後、同一画像を参照してMAX差分を制作。
6. 極ツ星10柱＋物語鍵神のみ横長カットインを制作。

通常絵とMAX絵を同時にゼロから生成しない。先に顔・衣装・象徴物を通常絵で固定し、MAX絵は承認済み通常絵を参照する。

## 9. 実装フェーズ

### Phase A — 再開と月送り

- 前回の灯。
- 家の火種三段目標。
- 次月の兆し。
- 月送り差分まとめ。

完了条件:

- 新規・中盤・断絶危機の三セーブで、ホームから10秒以内に次の行動を決められる。
- 休止による損失がない。
- 旧セーブを読み込める。

### Phase B — 帰還と神縁

- 今回変わったこと。
- 縁の三幕。
- 神縁の余韻。
- 神絵P0-A/P0-B投入。

完了条件:

- 帰還後の連続モーダルが一枚にまとまる。
- 同神との初回、縁3、縁MAXが見た目と文で区別できる。
- 欠落画像は全て安全に退避する。

### Phase C — 宿敵と世代

- 宿敵の傷跡。
- 世代の問い。
- 家譜への回収。
- 神絵P1/P2、選抜カットイン。

完了条件:

- 一度の敗北が次の対面で一文以上回収される。
- 世代の問いが次代で一度回収され、永久肥大しない。

## 10. 検証指標

テレメトリを導入する場合も、個人特定や通知誘導ではなく画面改善に限定する。

- 再開から最初の意味ある操作までの秒数。
- ホームで画面を往復して戻る回数。
- 出立準備中の取消理由。
- 月送り後に次の月も進めた割合。
- 一回のプレイで完了した小さな物語の数。
- 縁MAX到達神の偏り。
- `prefers-reduced-motion`、モバイル、キーボードでの完遂率。

継続率だけを成功指標にしない。**迷いの減少、物語の回収、気持ちよく止められること**も同格に扱う。
