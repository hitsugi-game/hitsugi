# 夢渡りの連作 — ビート表(2026-07-06 /story)

## 背骨
- **プレミス**: 一族が死を重ねるほど、家祖・汐里の夢に深く招かれ、千年前の真実——楽士と星喰いの物語——が一篇ずつ明かされる。
- **テーマ**: 看取りとは、憎むことをやめること。(最終戦「看取ってやれ!」への千年分の助走)
- **視点**: 夢を渡る「今代の子孫」(プレイヤーの一族)。汐里が語り、記憶が再生される。
- **欲求と欠落**: 一族の欲求=夜を明けさせる/欠落=なぜ夜が来たのかを知らない。連作は「知らないまま討つ」を「知った上で選ぶ」に変える。
- **対立の反転設計**: 玄冬を「討つべき怪物」→「哀しい空の子」へ、汐里を「聖なる犠牲者」→「愛した者を殺せなかったひとりの人」へ、7篇かけて反転する。
- **接続**: 夢1(既存DreamScene・無改変)=出会い。夢8の末尾が最終決戦(玄冬→汐里二段戦)と3結末(cut/save/inherit)の意味を予告する。

## 解禁設計(Hades型「死=物語の前進」)
- 発火: **看取り(死亡シーン)の後**、未読の解禁済みエピソードを1つだけ pendingScenes に積む(死→夢、の連鎖が主題)。
- unlock: deaths(一族の累計死者数)/ gen(到達世代)の **OR**(gossipと同型)。
- 既読管理: flags.dreamsSeen(string[])。既存 dreamSeen(夢1)は無改変。

## 成果物型・様式
- 実装データ: `src/core/data/dreams.ts`(DREAM_EPISODES: id/title/unlock/beats[])+store発火+DreamEpScene(既存DreamScene様式・cg_kiro背景)。
- 文体見本: 既存 DREAM_BEATS(Scenes.tsx)・lore.ts の縁起。一篇 9〜12ビート、地の文+「」の汐里の声。既存夢1と同じ「クリックで進む」朗読形式。
- 本文執筆: Fable直筆(7篇は量産規模でなく、既存文体との一貫性が品質の生命線のため。sonnet量産は文体揺れリスクで不採用と記録)。

## 品質ゲート予定
- 機械QA: tsc緑・unlock単調性・発火フロー実プレイ(死亡→夢)・既読の永続。
- 独立評価(opus別コンテキスト): 一貫性/因果連鎖/キャラ動機/テーマ表出/hook強度(各≥3・計≥18)。汎用物語レンズでありプロ編集者の代替ではない。
- provenance: 全要素オリジナル[事実(本作固有名詞・設定)]。俺屍からの物語借用なし(システムのインスパイアのみ=アイデア水準)。
