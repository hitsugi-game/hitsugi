# M50 適応型音楽・人物表示改善 実装正本

## 目的

既存のWeb Audio合成音を「短い同一ループ」から、画面・戦況・血脈に意味を持つ音楽へ組み替える。音を出せない環境でも攻略情報を失わせず、一族表示は横送りや炎記号に頼らず人物を見分けられる状態にする。

## 音楽ディレクション

- 全曲は平調子を共通語彙とし、`opening → theme → answer → breath → lineage → surge`の句を曲ごとに異なる順序で構成する。休符のある`breath`を必ず挟み、常時高密度にしない。
- 家祖IDの文字コード総和を五音へ写像した三音を`lineage`句へ挿入する。同じ家は画面が変わっても同じ血脈音を持つ。
- 通常戦、稀相、主戦は別曲とする。戦況は味方HP低下と敵HP低下を別の高揚要因として0〜1へ正規化し、旋律密度と打音だけを連続変化させる。
- 台詞・辞世・重要SEでは音楽busを短くduckし、文字の可読性と声なき物語の余白を優先する。

| 場面 | 曲 | 役割 |
|---|---|---|
| 題・導入 | 遠灯 / 命の間 | 汐里の子守唄を断片で予告する |
| 家・郷 | 家の座 | 血脈音を最も穏やかに反復する |
| 星契り | 天の逆歌 | 音域を上げ、未来を選ぶ緊張を作る |
| 鍛冶・普請 | 鉄の八拍 | 作業の手触りを短い打音で示す |
| 出立・遠征・Dungeon | 道なき灯 | 移動と探索を同じ旅の連続として結ぶ |
| 通常 / 稀相 / 主戦 | 灯刃 / 白金の脈動 / 常夜の底 | 遭遇価値を聴覚だけに依存せず区別する |
| 家譜・図鑑 | 名残の墨 | 収集を静かな振り返りにする |
| 結末 | 千年の岐路 | 子守唄と血脈音を回収する |

## 音響基盤

`music / effects / ambience`を独立gainへ分け、DynamicsCompressorを通るmasterへ合流する。画面遷移は旧曲と新曲を0.28〜1.15秒でcrossfadeし、schedulerは常に各系統1本以下とする。地域環境音も専用gainを0.38秒でfadeし、旧地域の長い余韻を新地域へ残さない。初回pointer/keyboard操作まで`AudioContext`を生成せず、非表示時は停止、復帰時は希望曲・希望環境音を再同期する。音楽・効果音・環境音・全体音量、消音、起伏を抑えるモードはlocalStorageへ安全に保存する。

第三者音源、外部作曲サービス、新規音声ファイルは使用しない。配信物は既存コードとブラウザ内合成だけで完結する。

## 一族UI・人物画像

- 小札は`auto-fit + minmax(248px, 1fr)`のgridへ変更し、PCでもmobileでも内部横スクロールを発生させない。
- 灯形未決定・元服前の人物には、属性に対応する既存配信中の顔絵を「稽古着の仮肖像」として表示する。成人の儀後は選んだ灯形の顔絵へ切り替える。
- `玄`を含む幼子が炎記号へ縮退するのは、既存顔絵の読込がすべて失敗した場合だけとする。新規画像の生成・権利台帳追加は行わない。
- `face_*`は既存公開版ですでに配信中だが、生成モデルまで遡るライセンス系譜は既存文書上未確認である。M50で「権利確認済み」へ昇格せず、同一ゲーム内での再利用に限定し、既存素材監査gateを継承する。

## 受入基準

1. 全23 screen IDが11曲のいずれかへ明示的に対応し、通常・稀相・主戦が分かれる。
2. gesture前はAudioContext未生成、遷移後もmusic schedulerは1本、消音で環境音0、復帰で1本へ戻る。
3. 設定に全体・音楽・効果音・環境音の4操作があり、PC二列/mobile一列で横overflowがない。
4. 一族小札は横送り不要で、灯形未決定の人物にも`img`が表示される。
5. 既存save schema、戦闘計算、報酬、全戦闘オートを変更しない。
6. 機械検証、PC1280/mobile390実画面、独立監査、Ship Check、Pages公開実測を通す。

## 外部評価gate

音楽の固有性・反復疲労・場面適合・重要文の読みやすさは、公開後に初見者を含む実試聴で確認する。コード上の構成差と動作合格だけで「魅力向上が実証された」とは扱わない。

既存`face_*`のモデルライセンス系譜確認も別gateとして残す。今回の公開は新規人物素材を追加せず、公開済みファイルのゲーム内再利用に限定する。
