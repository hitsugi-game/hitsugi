## 2. 中核転換①: 歩けるダンジョン(キャラクター操作)

### 2.1 探索仕様(俺屍のシンボルエンカウント継承)

- **描画**: PixiJS(WebGL)。俯瞰2D、タイルグリッド(32px)、キャラは4方向歩行スプライト
- **操作**: PC=WASD/矢印+スペース(調べる)、スマホ=仮想スティック+タップ。**オート探索補助**(目的地タップで経路移動)
- **シンボルエンカウント**: 敵影(提灯の灯り)がマップを徘徊。接触で戦闘。背後接触で先制/されたら不意打ち。**灯が減ると敵影が増え、速くなる**(v0.2の灯システムを空間化)
- **ダンジョン構造**: 各3〜7フロア。階段・宝箱・焚火(セーブ/回復)・仕掛け(一方通行の霧、崩れ橋、灯を消すと開く扉=リスク仕掛け)・隠し部屋(運が高いと視える)
- **帰り火**: どこでも使用可(v0.2継承)。深部ほど持ち帰り倍率UP
- **1回の出撃=1ヶ月**。ダンジョン内は実時間でなく「灯」で管理(現行踏襲)

### 2.2 マップ制作方式

手描きレイアウト(JSON)×プロシージャル装飾。1フロア=約40×40。エディタは作らず、テキストベースのマップ定義(ASCII → タイル変換)で量産。

### 2.3 戦闘の視覚化

- 現行ターン制エンジンを**そのまま流用**し、表示層をPixiJSへ:
  - 味方4人のスプライトが陣形(前衛/後衛)で並ぶ。敵はスプライト表示
  - 攻撃時: 踏み込み+斬撃エフェクト+ダメージポップ。術: パーティクル
  - **継足連携**: 家族が連続で繋ぐと画面端に「継」の火文字が灯っていく演出
- 戦闘テンポ: 1戦60〜90秒維持。倍速・オート続投

**総目標: 約2,000アセット**(現19枚の約100倍)。codexバッチ(stdin閉じ・15枚/セッション・照合コピー・自動圧縮)は検証済みの生産ラインとして確立している。
