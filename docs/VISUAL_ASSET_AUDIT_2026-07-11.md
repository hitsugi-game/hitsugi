# 全画面ビジュアル素材監査 2026-07-11

## 結論

プレイヤーが感じている「同じ画像が使われている」「マップやボスの姿が分からない」は、データ参照の重複より、**画像欠落時に同じ代替表示へ退避すること**と、**出立画面が選択前に画像を出さないこと**が主因である。

既存ファイルの画素ハッシュ監査では、神・敵・ボス627枚に完全一致画像は0組。神通常絵120枚のdHash近似一致も0組だった。つまり、既存画像の単純コピーではなく、追加コンテンツの画像が生成工程へ追随していない。

## 重複監査

- 神データのportrait参照重複: 0。
- 敵データのsprite参照重複: 0。
- `public/img` の神・敵・ボス627枚をデコード後の画素で比較: 完全一致0。
- 神通常絵120枚のdHash距離4以下: 0組。

したがって「同じ画像に見える」箇所は次を疑う。

1. `MaybeImg` が404時に画像を静かに消す。
2. 人物/アイコン系が同じ炎や絵文字へ退避する。
3. 40地域が内部ダンジョンでは4テーマしか使わず、プロップの種類も各テーマで反復する。
4. 未選択の出立画面は右側に画像を出さず、説明文だけになる。
5. 敵の若/老差分が未生成だと、戦闘上の種別差を絵で理解できない。

## 出立画面の構造問題

添付状態では地域ノードが灯っていても `selectedRegion === null` のため、右側は説明文だけ。画像はクリック後に初めて `MaybeImg` で表示される。未生成地域ならクリック後も画像が消える。

改善要件:

- 初期表示時に「現在行ける最前線」を自動選択する。
- 右側は常に16:9の地域画を表示する。未選択状態を作らない。
- 地域画へボスの黒い輪郭、異名、属性兆候、主討伐状態を重ねる。
- 物見櫓Lvに応じて、敵傾向・危険床・主の一手を段階的に開示する。
- 地図上は全40名称を同密度で並べず、現在地±2〜3地域を強く、他を絵巻の遠景へ下げる。
- ノード選択時は地域画を「灯写し」で0.35秒だけ明るく現す。散発的な常時アニメは増やさない。

## ダンジョンの構造問題

40地域に対し描画テーマは `forest / zaka / tani / miyama` の4系統。各テーマのプロップも10〜14種を2変種程度で回している。地名と説明が40あっても、内部体験は「4種類の暗い地形」に収束しやすい。

素材だけを増やしても解決しない。地域ごとに以下が必要。

- 署名ランドマーク1つ。
- 小プロップ3つ。
- 床/壁デカール1系統。
- 粒子または天候1系統。
- ボスへ続く痕跡3段階。
- 色・音・光の変化1組。

実装設計は `docs/VISUAL_RECOVERY_DUNGEON_PLAN.md` を正とする。
